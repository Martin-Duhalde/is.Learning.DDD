Documentation Directive
=======================

Purpose
-------
Provide guidance on how to maintain documentation for audits, guides, and AI directives.

Structure
---------
- Audits: `docs/audits/` (detailed findings, session caches).
- Guides: `docs/guides/` (roadmaps, architecture notes, agent instructions).
- Work Orders: `docs/work-orders/` (Kanban, detail files).
- AI Directives: `docs/ai-directives/` (this folder).

Maintenance Rules
-----------------
1. Use ISO date prefixes (`YYYY-MM-DD`) when creating new audit/guide files.
2. Reference related documents in each guide/audit to keep navigation easy.
3. Update checklists and roadmaps immediately after completing tasks.
4. When new methodology decisions are made, add or update an AI directive to reflect the process.

Expectations
------------
- Agents must read relevant directives before acting.
- Any new documentation format or directory should be recorded here.
